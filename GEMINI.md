## General Instructions

- When you generate new TypeScript code, follow the best practices for TypeScript development.
- Implement Atomic Design for components and ensure that your code is modular and reusable.
- Prefer functional programming paradigms where appropriate.
- Always favor readability and maintainability in your code.
- SOLID Principles and Clean Code/Architecture
- Prefer composition over inheritance
- Use interfaces as much as possible to avoid coupling and to promote flexibility.

## Coding Style

- Use 2 spaces for indentation.
- Prefix interface names with `I` (for example, `IUserService`).
- Always use strict equality (`===` and `!==`).
